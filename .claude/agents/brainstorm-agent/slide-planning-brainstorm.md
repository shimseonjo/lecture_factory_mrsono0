# 슬라이드 기획 브레인스토밍 (Phase 2) 세부 워크플로우

구성안이 **"무엇을 가르칠까(What)"**, 교안이 **"어떻게 가르칠까(How)"**를 정의했다면, 슬라이드 기획 Phase 2는 **"어떻게 시각화할까(How to visualize)"**를 브레인스토밍한다.
Assertion-Evidence 구조 설계, 시각화 아이디어 생성, 레이아웃 패턴 배정, 인터랙션 요소 구상이 핵심 산출물이다.

### 전체 흐름

```
Step 0: 입력 로드 + 시각화 브레인스토밍 계획 수립
  │     input_data.json + session 파일 시드 추출 → brainstorm_plan.md
  │
  ├── Step 1: 발산적 시각화 아이디어 생성 → divergent_ideas.md
  │   (4가지 발산 기법 × session 시드 × 4가지 카테고리)
  │
  ├── Step 2: 세션별 매핑 + 클러스터링 → idea_clusters.md
  │   (세션 귀속, GRR 단계, 슬라이드 유형, 도구 적합성)
  │
  ├── Step 3: 다관점 검증 → review_result.md
  │   (2개 역할 순차 시뮬레이션 + Decision Log)
  │
  ├── Step 4: AE 구조 + 레이아웃 + 인터랙션 구체화
  │   (검증 통과 아이디어를 구체적 AE 슬라이드/레이아웃/인터랙션으로 정교화)
  │
  └── Step 5: 통합 산출물 작성 → brainstorm_result.md ★
      (7개 섹션 구조화 문서)
```

### 산출물 목록

```
{output_dir}/
├── brainstorm_plan.md      # Step 0: 시각화 브레인스토밍 계획
├── divergent_ideas.md      # Step 1: 발산 아이디어 원시 목록
├── idea_clusters.md        # Step 2: 세션별 클러스터링
├── review_result.md        # Step 3: 다관점 검증 + Decision Log
└── brainstorm_result.md    # Step 5: 최종 통합 산출물 ★
```

### Step 0: 입력 로드 + 시각화 브레인스토밍 계획 수립

| 항목 | 내용 |
|------|------|
| 입력 | `{output_dir}/input_data.json`, `session_manifest[].file_path`의 session 파일들 |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/brainstorm_plan.md` |

**동작**:

1. `.claude/templates/input-schema-slide-planning.json` 읽기 — slide_config 각 필드의 enum 값, 의미, 필드 간 관계를 사전 이해
2. `input_data.json` 읽기 — 슬라이드 기획 특화 필드 추출:

   | input_data.json 필드 | 활용 용도 |
   |---------------------|----------|
   | `slide_config.slide_tool` | 인터랙션 기법 분기 (marp: Fragment 미지원 / slidev: v-click, Magic Move 지원) |
   | `slide_config.design_tone` | 시각화 방향 (friendly_visual→비유 시각화/professional→도표 중심) |
   | `slide_config.scope` | 대상 세션 필터링 (all/selected) |
   | `slide_config.estimated_slides` | 시간 관리자 검증 기준 |
   | `slide_config.slide_density_table` | GRR 단계별 밀도 검증 기준 |
   | `session_manifest[]` | 세션 순회 단위, file_path로 교안 로드, grr로 밀도 검증 |
   | `source_script.lecture_root` | session 파일 절대 경로 구성 |
   | `tone_examples` | 비유 기반 시각화 시드 |
   | `keywords` | 기술 키워드 → 코드 워크스루 대상 식별 |
   | `lab_environment` | 코드 교육 여부 판별, 슬라이드 도구 선택 근거 |

3. Session 파일 시드 추출 — `session_manifest`를 순회하며 각 session 파일 Read:

   | 시드 유형 | 추출 패턴 | 용도 |
   |----------|----------|------|
   | **발화문** | `> "..."` | Assertion 제목 후보 |
   | **코드 블록** | ` ```...``` ` | 코드 워크스루 대상 |
   | **발문** | `❓` 마커 또는 발문 테이블 | 인터랙션 슬라이드 후보 |
   | **비유/메타포** | 비유, 메타포 키워드 포함 문장 | 시각화 아이디어 후보 |
   | **활동 지시** | `[...]` 행동 지시 블록 | Activity 슬라이드 후보 |

4. 슬라이드 도구별 브레인스토밍 초점 설정:
   - `marp`: 정적 레이아웃 중심, Fragment 미지원 → 분절 슬라이드로 Progressive Disclosure 대체
   - `slidev`: v-click, Magic Move, 줄별 하이라이트 → 인터랙션 풍부, 코드 워크스루 최적
   - `gamma`: 시각적 완성도 중심, AI 자동 레이아웃 → 디자인 지시 비중 높음
   - `revealjs`: Fragment + 커스텀 애니메이션 → 최대 인터랙션

5. 세션 수 × 슬라이드 유형(12가지) = 브레인스토밍 조합 공간 산출
6. 적용할 발산 기법 선택 (4개 전부 적용)
7. `brainstorm_plan.md` 작성:
   - 시드 목록 (세션별 추출 시드 요약)
   - 선택된 발산 기법과 도구별 초점
   - design_tone별 시각화 방향
   - 검증 관점 정의
   - 브레인스토밍 단위 수 (세션 × 슬라이드 유형 카테고리)

### Step 1: 발산적 시각화 아이디어 생성 (슬라이드 기획 특화 4가지 기법)

| 항목 | 내용 |
|------|------|
| 입력 | `brainstorm_plan.md`, `input_data.json`, session 파일들 |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/divergent_ideas.md` |

#### 슬라이드 기획 특화 4가지 발산 기법

| # | 기법 | 적용 방법 | 생성 대상 |
|---|------|----------|----------|
| 1 | **AE 변환 (Assertion-Evidence)** | 발화문에서 핵심 주장 추출 → 12단어 이내 Assertion 제목으로 변환 → 적합한 시각 증거 유형 구상 (도표/코드/이미지/다이어그램). 불릿포인트 대신 시각 증거로 주장 뒷받침 | 슬라이드 제목 + 시각 증거 쌍 |
| 2 | **6W 시각화 매핑** | 콘텐츠를 6W(Who/What/When/Where/How/Why)로 분류 → Dan Roam의 SQVID 프레임으로 최적 시각화 유형 배정. Who→조직도/관계도, What→블록다이어그램, When→타임라인, Where→맵/흐름도, How→플로우차트, Why→인과관계도 | 시각화 유형 + 레이아웃 패턴 |
| 3 | **범위 전환 (Macro↔Micro)** | 매크로(전체 아키텍처/프로세스 개요) ↔ 마이크로(단일 코드 라인/세부 개념) 간 시점 전환. 동일 내용을 "숲"과 "나무" 관점으로 분절하여 정보 밀도 변주 | 정보 밀도 변주, 분절 전략 |
| 4 | **인터랙션 설계** | slide_tool의 기능별 인터랙션 매핑. Progressive Disclosure(단계적 공개), 코드 하이라이트(줄별/블록별), 전환 애니메이션, 비교 토글. 도구별 지원 기능에 따른 대체 전략 포함 | Progressive Disclosure, 코드 하이라이트, 전환 효과 |

#### SCAMPER 보조 도구 (4가지 발산 기법 적용 시 보조)

| SCAMPER | 시각화 적용 질문 |
|---------|--------------|
| **S**ubstitute | "불릿포인트를 어떤 시각 요소로 대체할 수 있는가?" |
| **C**ombine | "두 개념을 하나의 비교 슬라이드로 결합하면?" |
| **A**dapt | "다른 분야의 시각화 방식을 이 개념에 차용하면?" |
| **M**odify | "정보 밀도를 높이거나 낮추면 어떤 레이아웃이 적합한가?" |
| **P**ut to another use | "이 다이어그램을 퀴즈 슬라이드로도 활용하면?" |
| **E**liminate | "이 슬라이드에서 제거 가능한 시각 요소는?" |
| **R**everse | "결과→과정 역순으로 보여주면 어떤 효과가 있는가?" |

#### 동작

1. session 파일에서 추출한 시드를 기반으로, 4가지 발산 기법을 순차 적용
2. **4가지 카테고리별** 아이디어 생성:
   - **시각화 아이디어**: 6W 분류별 최적 시각화 유형 (도표/다이어그램/이미지/코드 등)
   - **레이아웃 패턴**: 12가지 슬라이드 유형 중 적합한 유형 배정 + 구체적 배치 구상
   - **인터랙션 요소**: slide_tool별 적용 가능한 인터랙션 기법 (v-click, Fragment, 하이라이트 등)
   - **AE 구조**: Assertion 제목 + Evidence 유형 쌍
3. 각 아이디어 기록 형식:
   ```
   ### 아이디어 {번호}: {제목}
   - 카테고리: {시각화/레이아웃/인터랙션/AE구조}
   - 설명: {1~2문장}
   - 발산 기법: {사용한 기법명}
   - 시드: {관련 session 시드 유형 + 원문 요약}
   - 관련 세션: {D1-1, D2-3, ...}
   - GRR 단계: {I Do/We Do/You Do}
   - 슬라이드 유형: {12유형 중}
   ```
4. 의도적 과잉 생성: 최종 필요량의 2~3배 (후속 Step에서 선별)
5. 평가 유보: 이 단계에서는 품질 판단을 하지 않음
6. `divergent_ideas.md` 작성

### Step 2: 세션별 매핑 + 클러스터링

| 항목 | 내용 |
|------|------|
| 입력 | `divergent_ideas.md`, `input_data.json` |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/idea_clusters.md` |

#### 클러스터링 기준 (슬라이드 기획 특화 4가지)

- **세션 귀속**: 어떤 session_id에 적용할 수 있는 아이디어인가
- **GRR 단계 귀속**: I Do(시범) → We Do(안내) → You Do(독립) — GRR 단계별 시각화 밀도 차이 반영
- **슬라이드 유형 귀속**: 12가지 유형 중 어디에 해당하는가
- **도구 적합성**: slide_tool에서 구현 가능한가 (marp 대체 방안 포함)

#### GRR 단계별 시각화 밀도 원칙

| GRR 단계 | 시각화 밀도 | 슬라이드 특성 |
|---------|-----------|------------|
| I Do (시범) | **높음** — 풍부한 시각 자료 | 개념 설명, 코드 워크스루, 다이어그램 중심 |
| We Do (안내) | **중간** — 참여형 시각 | 발문 슬라이드, 부분 공개, 힌트 카드 |
| You Do (독립) | **낮음** — 최소 시각 | 과제 지시, 체크리스트, 결과 템플릿 |
| 도입/정리 | **중간** — 맥락 제공 | 아젠다, 요약, 전환 슬라이드 |

#### 동작

1. 발산 아이디어를 세션별 × GRR 단계별로 매핑
2. 슬라이드 유형별 그룹화 및 중복 제거
3. 도구 적합성 필터링:
   - `marp`: Fragment/v-click 아이디어 → 분절 슬라이드로 대체 방안 기록
   - `slidev`: 모든 인터랙션 유형 사용 가능
4. 클러스터 요약 테이블:
   ```
   | 세션 | GRR 단계 | 슬라이드 유형 | 아이디어 수 | 도구 적합성 | 예상 장 수 |
   |------|---------|------------|-----------|-----------|----------|
   ```
5. 세션별 아이디어 수 집계 (장수 확정은 Phase 3 architecture-agent에서 수행)
6. `idea_clusters.md` 작성

### Step 3: 다관점 검증 (2개 역할)

| 항목 | 내용 |
|------|------|
| 입력 | `idea_clusters.md`, `input_data.json` |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/review_result.md` |

교안에서 이미 SLO 정렬, Bloom's 적합성, Gagne 커버리지가 확정되어 있으므로, 슬라이드 기획 단계에서는 **시각적 인지 부하 + 슬라이드 수 현실성**만 검증한다.
단일 에이전트가 순차적으로 각 관점을 시뮬레이션한다 (Agent 중첩 금지 제약).

#### 슬라이드 기획 도메인 역할 정의

**1. 학습자 대변인 (Learner Advocate)**

| 항목 | 내용 |
|------|------|
| 역할 | 시각적 인지 부하, 메시지 즉시 파악 가능성, 정보 밀도 적절성 검증 |
| 핵심 질문 | "3초 내 핵심 메시지를 파악할 수 있는가?", "인지 부하가 과도한 슬라이드는?", "시각 요소가 학습을 돕는가 방해하는가?" |
| 검증 기준 | One Idea Rule (1슬라이드 1개념), Mayer 멀티미디어 원칙 (일관성·시간근접·중복 등), 6×6 규칙 (6줄 이하, 줄당 6단어 이하), AE 적용률 (Assertion 제목 + 시각 증거 쌍 비율) |
| 금지 | 교안 콘텐츠 변경 제안 금지 — 시각적 표현 관점에서만 평가 |

**2. 시간 관리자 (Time Guardian)**

| 항목 | 내용 |
|------|------|
| 역할 | GRR 밀도 균형, 아이디어 편중 여부 정성 검증 |
| 핵심 질문 | "GRR 단계별 아이디어가 밀도 원칙(I Do>We Do>You Do)에 부합하는가?", "특정 세션에 아이디어가 극단적으로 편중되어 있지 않은가?", "아이디어 부족으로 빈 GRR 구간이 있지 않은가?" |
| 검증 기준 | GRR 밀도 원칙 부합(I Do 풍부 > We Do 중간 > You Do 최소), 세션 간 아이디어 분포 균형, 12유형 중 최소 3유형 이상 포함 |
| 금지 | 콘텐츠 우선순위 변경 금지, **정량적 장수 확정 금지** — 장수 결정은 Phase 3 architecture-agent에서 수행 |

#### 동작

1. 관점 1~2를 순차 시뮬레이션 — 각 관점별 섹션으로 기록
2. 마무리: 충돌 해결 + 판정 태그 부여
   - `ADOPT`: 그대로 채택
   - `REVISE`: 수정 후 채택 (수정 방향 명시)
   - `DROP`: 폐기 (근거 명시)
   - `MERGE`: 다른 아이디어와 병합
3. Decision Log 작성:
   ```
   | # | 판정 | 대상 아이디어 | 수정 방향/병합 대상 | 이유 |
   |---|------|------------|-------------------|------|
   ```
4. `review_result.md` 작성

### Step 4: AE 구조 + 레이아웃 + 인터랙션 구체화

| 항목 | 내용 |
|------|------|
| 입력 | `review_result.md`, `input_data.json`, `brainstorm_plan.md` |
| 도구 | Read, Write |
| 산출물 | (Step 5에서 brainstorm_result.md에 통합) |

#### 4-1. Assertion-Evidence 구조 구체화

검증 통과한 AE 아이디어를 세션별 GRR 단계별로 정교화한다.

**AE 슬라이드 구조**:
- **Assertion (제목)**: 12단어 이내의 완전한 주장문 (토픽 라벨이 아닌 선언형 문장)
- **Evidence (본문)**: 시각 증거 — 도표, 코드, 이미지, 다이어그램 등 (불릿포인트 최소화)

**GRR 단계별 AE 적용 비율**:

| GRR 단계 | AE 적용률 | 근거 |
|---------|----------|------|
| I Do | ≥80% | 핵심 개념 전달이 목표 → 명확한 주장+증거 필수 |
| We Do | ≥60% | 참여형 → 일부 발문/활동 슬라이드는 비AE 허용 |
| You Do | ≥30% | 과제 지시 중심 → AE보다 실습 지시가 중심 |
| 도입/정리 | 선택적 | 아젠다/요약은 목록형 허용 |

#### 4-2. 레이아웃 패턴 배정

검증 통과한 아이디어를 12가지 슬라이드 유형에 배정하고, 구체적 레이아웃을 설계한다.

**12가지 슬라이드 유형**:

| # | 유형 | 용도 | 전형적 레이아웃 |
|---|------|------|-------------|
| 1 | 제목 | 섹션/세션 시작 | 대형 타이포 + 배경 이미지 |
| 2 | 아젠다 | 세션 구조 소개 | 번호 목록 + 현재 위치 하이라이트 |
| 3 | 섹션전환 | GRR 단계/토픽 전환 | 큰 텍스트 + 아이콘 |
| 4 | 개념설명 | I Do 핵심 개념 | AE: 주장 제목 + 시각 증거 |
| 5 | 코드 | 코드 워크스루 | 코드 블록 + 줄별 하이라이트 |
| 6 | 비교 | A vs B 대조 | 2분할 레이아웃 |
| 7 | 데이터+인사이트 | 수치/차트 기반 | 차트 + 핵심 수치 + 해석 |
| 8 | 이미지 | 시각적 개념 전달 | 전체 이미지 + 오버레이 텍스트 |
| 9 | 실습/활동 | We Do/You Do 지시 | 과제 설명 + 체크리스트 + 제한시간 |
| 10 | 퀴즈/발문 | Bloom's 질문·선택지 | 질문 + 선택지(해당 시) |
| 11 | 타임라인 | 프로세스/역사 | 수평/수직 단계도 |
| 12 | 핵심요약 | 정리/복습 | 3~5항목 요약 카드 |

#### 4-3. 인터랙션 요소 구체화

검증 통과한 인터랙션 아이디어를 slide_tool별 구현 명세로 정교화한다.

**도구별 인터랙션 기능 매핑**:

| 인터랙션 | slidev | marp | revealjs | gamma |
|---------|--------|------|----------|-------|
| Progressive Disclosure | `v-click` | ❌ → 분절 슬라이드 | `fragment` | ❌ → 분절 |
| 코드 줄별 하이라이트 | `{lines}` 속성 | ❌ → 주석 하이라이트 | `data-line-numbers` | ❌ |
| Magic Move (코드 변환) | `<<< @/` | ❌ → 전후 비교 슬라이드 | ❌ → 전후 비교 | ❌ |
| 발표자 노트 | `<!-- 노트 -->` | `<!-- 노트 -->` | `<aside class="notes">` | 별도 패널 |
| 2분할 레이아웃 | `::left::` `::right::` | CSS `columns` | `<div class="fragment">` | 자동 |

**marp 대체 전략**: marp에서 지원하지 않는 인터랙션은 동등 효과의 정적 방법으로 대체:
- v-click → 동일 콘텐츠를 단계적으로 추가한 분절 슬라이드 시퀀스
- 줄별 하이라이트 → 주석(`// ← 여기!`)이나 화살표 이미지
- Magic Move → Before/After 2장 슬라이드

#### 4-4. 코드 워크스루 패턴 (5가지)

코드 교육 콘텐츠에 적용할 5가지 코드 워크스루 패턴:

| # | 패턴 | 설명 | 적합한 상황 |
|---|------|------|-----------|
| 1 | **줄별 해설** | 코드 줄마다 하이라이트 이동 + 해설 | 새로운 API/문법 소개 |
| 2 | **빌드업** | 빈 파일 → 점진적 코드 추가 | 처음부터 작성 과정 시연 |
| 3 | **Before→After** | 리팩토링 전후 비교 | 코드 개선/패턴 적용 |
| 4 | **에러→수정** | 오류 코드 → 디버깅 → 수정 | 오개념 교정, 디버깅 교육 |
| 5 | **확대경** | 전체 코드 → 특정 부분 확대 | 대규모 코드베이스 탐색 |

#### 4-5. Mayer 멀티미디어 원칙 적용 가이드

슬라이드 설계에 적용할 Mayer 멀티미디어 학습 8원칙:

| # | 원칙 | 효과크기(d) | 슬라이드 적용 지침 |
|---|------|-----------|----------------|
| 1 | **일관성** (Coherence) | 1.32 | 학습 무관 장식 요소 제거, SNR(신호 대 소음) 최대화 |
| 2 | **시간근접** (Temporal Contiguity) | 1.30 | 설명과 시각 자료를 동일 슬라이드에 배치 |
| 3 | **신호화** (Signaling) | 0.52 | 핵심 정보 시각적 강조 (색상, 크기, 화살표) |
| 4 | **공간근접** (Spatial Contiguity) | 1.12 | 텍스트와 관련 이미지를 인접 배치 |
| 5 | **중복** (Redundancy) | 0.86 | 발화문을 슬라이드에 그대로 복제하지 않음 |
| 6 | **분절화** (Segmenting) | 0.98 | 복잡한 내용을 여러 슬라이드로 분절 |
| 7 | **사전훈련** (Pre-training) | 0.85 | 핵심 용어/개념을 먼저 소개하는 슬라이드 선행 |
| 8 | **멀티미디어** (Multimedia) | 1.39 | 텍스트만이 아닌 텍스트+시각 자료 결합 |
