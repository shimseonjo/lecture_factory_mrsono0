# 거시적 강의구성안 작성 (Phase 8)

## 설계 원칙

1. **요약-변환 전용**: 새로운 정보를 생성하지 않는다. `lecture_outline.md`의 데이터만 추출·병합·요약한다.
2. **한 페이지 테이블**: 과정 전체를 조감할 수 있는 단일 테이블 문서로 출력한다.
3. **교시→시간 블록 병합**: 50분 교시 단위를 시간(hr) 블록으로 병합하여 모듈 단위로 표현한다.

---

## 전체 흐름

| Step | 역할 | 입출력 |
|------|------|--------|
| Step 0 | 입력 로드 + 파싱 | Read: lecture_outline.md, input_data.json, 템플릿 |
| Step 1 | 상위 메타 정보 추출 | 제목, 과정 목표, 기대 효과, 실습 환경 |
| Step 2 | 교시→모듈 병합 | 시간 블록 계산 + 카테고리/태그 매핑 |
| Step 3 | 테이블 조립 + 출력 | Write → macro_outline.md |

---

## Step 0: 입력 로드

**Read 대상**:
1. `{output_dir}/lecture_outline.md` — 다음 섹션을 파싱:
   - §1-1 기본 정보 (강의명, 실습 환경)
   - §1-2 시간 예산 (총 일수, 일일 교육 시간)
   - §1-4 일별 시간표 (각 Day의 시작/종료 시간, 점심 시간)
   - §3-1 CLO (기대 효과 도출용)
   - §5-1 일별 테마 개요
   - §5-3 Day별 차시 상세 (교시 테이블: 교시 번호, 주제, content_type, 수업방법, 학습활동)
2. `{output_dir}/input_data.json` — schedule, pedagogy, lab_environment 필드
3. `.claude/templates/macro-outline-template.md`

## Step 1: 상위 메타 정보 추출

| 거시적 구성안 항목 | 소스 | 변환 규칙 |
|----------------|------|----------|
| 제목 | §1-1 강의명 + input_data.json schedule.days | `{강의명} ({N}일 과정)` 형식으로 축약 |
| 과정 목표 | §1-1 기본 정보의 강의명 필드 | 핵심 목적을 1~2문장으로 요약 |
| 기대 효과 | §3-1 CLO | 각 CLO를 **수강생 관점 효과**로 변환 (3~4개) |
| 과정 특징 | input_data.json pedagogy | 핵심 교수 전략 1~2문장 (있는 경우만) |
| 실습 환경 | input_data.json lab_environment | 핵심 도구/플랫폼 나열 (있는 경우만) |

**기대 효과 변환 예시**:
- CLO: "Spring Boot의 3-Tier 아키텍처를 설명할 수 있다" → 효과: "웹 애플리케이션의 계층 구조를 이해하고 설계할 수 있는 역량 확보"

## Step 2: 교시→모듈 병합

### 2-1. 병합 알고리즘

§5-3의 Day별 차시 테이블을 순회하며 연속 교시를 모듈로 병합한다.

**병합 판단 기준** (우선순위 순):
1. **SLO 공유**: 연속 교시가 동일 SLO를 공유하면 병합
2. **상위 주제 동일**: SLO가 다르더라도 주제명의 상위 개념이 같으면 병합 (예: "Controller 안내" + "Controller 독립")
3. **이론+실습 세트**: 같은 주제의 concept → hands-on 연속은 병합 허용 (카테고리: `[이론+실습]`)

**병합하지 않는 경우**:
- content_type이 다르고 주제도 다른 경우
- 아침 루틴(복습), 회고, 오리엔테이션 등 독립 활동 교시
- 점심시간을 사이에 둔 교시

### 2-2. 시간 계산

```
모듈 시간(hr) = 병합된 교시 수 × 50분 / 60
```

- 0.5 단위로 반올림 (예: 1.67 → 1.5, 2.5 → 2.5, 0.42 → 0.5)
- 반교시(25분)가 포함된 경우 실제 분 수 기준으로 계산
- 모든 일차의 모듈 시간 합 = 일일 교육 시간 (점심 제외)

### 2-3. 카테고리 매핑

| content_type | 수업방법 패턴 | 카테고리 |
|-------------|-------------|---------|
| concept | 강의* | [이론] |
| concept | 비교* | [비교분석] |
| hands-on | 안내 실습, 독립 실습 | [실습] |
| hands-on | 프로젝트* | [프로젝트] |
| hands-on | 탐구* | [탐구] |
| activity | 정리, 복습 | [정리] |
| activity | 오리엔테이션, 시연 | [마인드셋] |
| activity | 발표 | [발표] |
| activity | 회고 | [회고] |
| 혼합 (concept+hands-on) | - | [이론+실습] |

카테고리가 명확하지 않으면 §5-1 일별 테마를 참고하여 판단한다.

### 2-4. 교수법 태그 매핑

| 패턴 | 태그 |
|------|------|
| AI 프롬프트/도구 활용 교시 | *생성형 AI 활용* |
| hands-on 실습 중심 | *프로세스 경험* |
| AI + 독립 실습 | *Vibe Coding + 검증* |
| 비유/메타포 기반 설명 | *비유 기반 설명* |
| 비교 시연 | *비교 시연* |
| 프로젝트/발표 기반 | *프로젝트 기반 학습* |
| 실무 데이터/사례 활용 | *실무 예제* |
| 노코드/로우코드 도구 | *노코드/로우코드* |

- 모듈에 여러 교시가 병합된 경우, 대표 태그 1~2개를 쉼표로 연결
- 태그가 해당되지 않는 경우 생략 가능

### 2-5. 교육 내용 생성

교육 내용 셀 패턴: `**[카테고리] 제목** - 설명 > *교수법 태그*`

- **제목**: 병합된 교시들의 주제를 상위 수준으로 요약 (간결한 1구)
- **설명**: 각 교시의 핵심 학습활동에서 키워드를 추출하여 하이픈(-)으로 연결 (2~4개)
- **교수법 태그**: §2-4 매핑표에서 결정

---

## Step 3: 테이블 조립 + 출력

### 3-1. 테이블 구성 규칙

1. 각 일차 첫 행의 모듈 열에만 `**{N}일차** ({시작}~{종료})` 표시
2. 같은 일차의 이후 행은 모듈 열을 비움
3. 시간 열: hr 단위 (소수점 1자리, 0.5 단위 반올림)
4. Day 순서대로 행 나열

### 3-2. 점심시간 표기

- 테이블 아래 `---` 구분선 후 이탤릭으로 점심시간 표기
- 예: `*1~5일차 점심시간: 12:00 ~ 13:00*`
- 일차별 시간이 다른 경우 예외 표기: `(단, 1일차는 13:00~14:00)`

### 3-3. 산출물 출력

`{output_dir}/macro_outline.md`에 Write한다.

## 금지 사항

1. `lecture_outline.md`에 없는 새로운 모듈/주제/시간 배치 창작 금지
2. 교시 병합 시 원본 시간 합계를 초과하는 시간 배정 금지
3. 교시 순서 변경 금지
4. 상세 가이드(도입/전개/정리 세그먼트) 정보 포함 금지 — 미시적 구성안의 영역
5. CLO/SLO/EQ 테이블, 평가 체계, 정렬 맵 등 상세 정보 포함 금지
6. 외부 검색(WebSearch, WebFetch) 금지
7. Agent 중첩 금지
