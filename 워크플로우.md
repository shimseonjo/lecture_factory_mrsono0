# 워크플로우

## 전체 파이프라인

```
/lecture-outline  →  /lecture-script   →  /slide-planning    →  /slide-generation
  강의구성안(7단계)     강의교안(7단계)      슬라이드 기획(5단계)    슬라이드 생성(3단계)
```

각 워크플로우는 독립 실행 가능하며, 이전 단계의 산출물을 입력으로 참조합니다.

---

## 강의구성안 (`/lecture-outline`) — 7단계

| Phase | 단계 | 에이전트 | 핵심 작업 |
|-------|------|---------|----------|
| 1 | 입력 수집 | input-agent | 필수 6개(Q1~Q6) + 선택 7개(Q7~Q12) → `input_data.json` |
| 2 | 탐색적 리서치 | research-agent | 참고자료 분석(로컬 + NotebookLM) + 트렌드, 유사 강의 (고착 효과 방지 필터 적용) |
| 3 | 브레인스토밍 | brainstorm-agent | 하위 주제 도출, 페르소나, 핵심 질문, Bloom's 매핑, 우선순위 분류 |
| 4 | 심화 리서치 | research-agent | 브레인스토밍 기반 사례 수집, 참고자료 심화 분석, 보충 콘텐츠 |
| 5 | 아키텍처 설계 | architecture-agent | Backward Design, 정렬 맵, 차시 구조, 시간 배분 |
| 6 | 구성안 작성 | writer-agent | `outline-template.md` 기반 최종 문서 생성 (GAIDE 5단계) |
| 7 | 품질 검토 | review-agent | QM Rubric 기반 36개 항목 검증 → PASS / CONDITIONAL PASS / REVISION REQUIRED |

**산출물**: `lectures/YYYY-MM-DD_{강의명}/01_outline/lecture_outline.md`

---

## 강의교안 (`/lecture-script`) — 7단계

| Phase | 단계 | 에이전트 | 핵심 작업 |
|-------|------|---------|----------|
| 1 | 입력 수집 | input-agent | 구성안 로드, 교수법 선택 |
| 2 | 탐색적 리서치 | research-agent | 교수법 사례, 유사 교안 벤치마킹 |
| 3 | 브레인스토밍 | brainstorm-agent | 발문 설계, 학습활동 아이디어, 실생활 사례 |
| 4 | 심화 리서치 | research-agent | 예시 자료, 보충 콘텐츠, 참고 문헌 |
| 5 | 교안 구조 설계 | architecture-agent | 도입-전개-정리, Gagné 9사태 적용 |
| 6 | 교안 작성 | writer-agent | `script-template.md` 기반 작성 |
| 7 | 품질 검토 | review-agent | 목표-활동-평가 정렬, 시간 배분, 용어/톤 일관성 |

**산출물**: `lectures/YYYY-MM-DD_{강의명}/02_script/lecture_script.md`

---

## 슬라이드 기획 (`/slide-planning`) — 5단계

| Phase | 단계 | 에이전트 | 핵심 작업 |
|-------|------|---------|----------|
| 1 | 입력 수집 | input-agent | 교안 로드, 슬라이드 도구 선택 |
| 2 | 브레인스토밍 | brainstorm-agent | 시각화 아이디어, 레이아웃 패턴 |
| 3 | 구조 설계 | architecture-agent | 슬라이드 수, 유형 배정, 시간 배분 |
| 4 | 기획안 작성 | writer-agent | `slide-plan-template.md` 기반 작성 |
| 5 | 품질 검토 | review-agent | 정보 밀도, 시각 계층, 학습목표 정렬 |

**산출물**: `lectures/YYYY-MM-DD_{강의명}/03_slide_plan/slide_plan.md`

---

## 슬라이드 생성 (`/slide-generation`) — 3단계

| Phase | 단계 | 에이전트 | 핵심 작업 |
|-------|------|---------|----------|
| 1 | 입력 수집 | input-agent | 기획안 로드, 출력 형식 선택(Marp/Slidev/Gamma) |
| 2 | 프롬프트 생성 | writer-agent | `slide-prompt-template.md` 기반 작성 |
| 3 | 품질 검토 | review-agent | 형식 검증, 콘텐츠 정확성, 일관성 |

**산출물**: `lectures/YYYY-MM-DD_{강의명}/04_slides/slides.md`

---

## 참조 문서

- **상세 가이드**: `.claude/Lecture_Creation_Guide.md`
- **시스템 규칙**: `.claude/CLAUDE.md` (산출물 저장 규칙)
- **각 워크플로우 상세**: `.claude/skills/{skill-name}/SKILL.md`
- **각 에이전트 상세**: `.claude/agents/{agent-name}/AGENT.md`
